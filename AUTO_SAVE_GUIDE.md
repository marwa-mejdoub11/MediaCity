## 🔄 Auto-Save Explained

### What Gets Auto-Saved?
1. **Media Title** - Immediately on edit
2. **Media Type** - Immediately on edit
3. **Availability Status** - Immediately on toggle
4. **Timestamps** - Auto-generated by Firestore
   - `createdAt`: When media was created
   - `updatedAt`: Last modification time

### How It Works
```
User Action → Validation → Firestore Write → Stream Update → UI Refresh
```

**Timeline:**
- Add media: ~500ms to sync
- Edit media: ~300ms to sync
- Delete media: ~300ms to sync
- Real-time listeners: Updates within 100ms

### What Happens if App Closes?
- ✅ All data already in Firestore
- ✅ No data loss
- ✅ Resume from any device with same account

## 🚨 Common Issues & Solutions

### Issue: "Initializing database setup"
**Cause**: Firestore initializing for first time
**Solution**: Wait 1-2 minutes, refresh app

### Issue: "Permission denied" when saving
**Cause**: Firestore rules not set correctly
**Solution**: 
```javascript
// Update rules to allow authenticated users:
match /users/{uid} {
  allow read, write: if request.auth.uid == uid;
  match /media/{document=**} {
    allow read, write: if request.auth.uid == uid;
  }
}
```

### Issue: "User not authenticated" error
**Cause**: User tried to add media without logging in
**Solution**: Ensure user completes login before adding media

### Issue: Data shows in Firestore but not in app
**Cause**: User ID mismatch or query filter
**Solution**: 
- Check if current user UID matches Firestore document path
- Verify Firestore rules grant read access

### Issue: Multiple devices show different data
**Cause**: Different user accounts or offline sync
**Solution**:
- Ensure using same Firebase account
- Enable offline persistence (auto-enabled)
- Force refresh by pulling down in app

## 📱 Device Sync Behavior

### Real-time Devices:
If you login on 2 devices at same time:
1. Device A adds media → Firestore
2. Device B sees update in ~100ms
3. Both devices show same data

### Offline Devices:
If Device A goes offline:
1. App buffers changes locally
2. Changes sync when connection restored
3. No data loss

### Conflict Resolution:
If both devices edit same media offline:
1. Last write wins (Firestore default)
2. Server timestamp used as tiebreaker
3. Both changes merge safely

## 🎯 Auto-Save Endpoints

### Add Media
```
POST users/{userId}/media
Body: {
  title: string,
  type: string,
  available: boolean,
  createdAt: timestamp,
  updatedAt: timestamp
}
```

### Update Media
```
PATCH users/{userId}/media/{mediaId}
Body: {
  title?: string,
  type?: string,
  available?: boolean,
  updatedAt: timestamp
}
```

### Delete Media
```
DELETE users/{userId}/media/{mediaId}
```

### Get Media
```
GET users/{userId}/media?orderBy=createdAt&direction=desc
Returns: List<Media> with real-time updates
```
